# 1 引言

编译器、汇编器以及类似的工具生成处理器执行的所有二进制代码。因此，这些工具在安全分析和加固相关二进制代码方面扮演着重要角色。

通常，保护所有特定安全加固方法的二进制文件的唯一实际方法就是让编译器来完成。随着近年来软件安全性的日益重要，编译器中实现的安全加固特性和缓解措施种类不断增多，这并不令人惊讶。确实，与几十年前相比，今天的编译器开发者更有可能实现安全特性而不是不实现。

此外，随着实现的技术范围不断扩展，要全面理解典型编译器中实现的所有安全特性非常困难。

这提出了一个实际问题：编译器开发者必须能够工作在安全加固特性上，但很难获得对这些编译器特性的良好、基本理解。

有很多材料解释了单个漏洞或攻击向量。也有很多演示解释了特定的利用。但似乎只有有限的材料提供了对所有漏洞和针对代码生成器在保护中扮演角色的攻击的有序概述。

本书旨在提供这样的结构化、广泛的概述。它并不一定深入到全部细节，而是旨在对所有相关高级攻击、漏洞、缓解和加固技术的各个方面进行详尽的描述。对于更详细的信息，本书提供了指向更具体技术细节材料的指针。

这本书的目的是为每一位需要了解与编译器相关的软件安全性的编译器开发者提供指南。尽管本书的重点是针对编译器开发者，但我们期望这本书对从事其他底层软件开发工作的人也会有所帮助。

## 1.1 为什么是开源书籍？

这本书的灵感源于找不到关于这个主题的良好概述的挫败感。Kristof Beyls 和 Georgia Kouveli，两位都在编译器安全特性方面工作的编译器工程师，都希望有这样一本书存在。在找不到这样的书后，他们决定尝试自己编写一本书。他们很快意识到，他们自己并不具备完成这项艰巨任务所需的所有专业知识。因此，他们决定以开源的方式尝试编写这本书，寻求众多专家的贡献。

当您阅读这本书时，它仍然是不完整的。这本书可能永远不会完成，因为新的漏洞仍在定期被发现。我们的希望是，将这本书作为一个开源项目来开发，将允许它继续发展和改进。这本书的开放源代码开发过程增加了它在出现新的漏洞和缓解措施时保持相关性的可能性。

Kristof 和 Georgia，最初的作者，并不是所有可能漏洞的专家。那么，如何获得高质量的内容来涵盖所有相关主题的计划是什么？有两个方面。

首先，通过研究特定主题，他们希望获得足够的知识来为这本书编写一份良好的总结。

其次，他们非常欢迎并邀请贡献。如果您对可能贡献内容感兴趣，请访问开源项目的首页 [`github.com/llsoftsec/llsoftsecbook`](https://github.com/llsoftsec/llsoftsecbook)。
