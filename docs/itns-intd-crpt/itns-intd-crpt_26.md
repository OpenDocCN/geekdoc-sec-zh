# 密码学的伦理、道德和政策维度

> 原文：[`intensecrypto.org/public/lec_24_policy.html`](https://intensecrypto.org/public/lec_24_policy.html)

*发现任何错误/打字错误/令人困惑的解释？[在 GitHub 上打开一个 issue](https://github.com/boazbk/crypto/issues/new)。您也可以在下面评论*

**★ 另请参阅本章的[PDF 版本**](https://files.boazbarak.org/crypto/lec_24_policy.pdf)（更好的格式/参考文献）★

这将不会是一次讲座，而是一次关于密码学引发的一些问题的讨论。我希望你阅读以下一些资料（也许还有其他资料）并思考以下问题：

讨论通常被框架为权衡隐私与安全，但我鼓励你批判性地看待这两个问题。尝试将当前情况与历史过去以及一些理想化的期望世界进行比较，通常是有教育意义的。在更广泛的环境中考虑密码学也是值得的。在支持监管和反对监管的阵营中，有些人过分解读了密码学的作用。

一方面，密码学可能不会带来加密无政府主义者宣言中所希望的“加密无政府主义”制度。例如，除了比特币的增长之外，我们看到了人们从现金转向信用卡和其他更易追踪且*不那么*匿名的支付方式（有趣的是，这些支付方式通常是由密码学所启用）。另一方面，尽管政府机构提出了“黑暗”的担忧，但收集大量数据并在搜查令友好的服务器上存储数据的强大商业激励仍然存在。显然，技术正在改变个人之间的关系，以及个人与大型组织和国家之间的关系。密码学是这些技术的一个重要组成部分，但不是唯一的，更重要的是，技术最终*被使用*的方式往往更多地与社会和商业因素有关，而不是技术本身。

所有这些话，重大的变化往往带来非同寻常的危险，因此，重要的是要有一个知情且合理的讨论，讨论密码学如何帮助或损害公共利益和私人利益。

一些值得考虑的问题是：

+   私下交流是否是一项基本的[人权](http://www.un.org/en/universal-declaration-human-rights/)？它是否应该扩展到远程交流？这应该是绝对的隐私，即使有法律搜查令也不能侵犯吗？如果只有通过法律搜查令才能实施窃听，这在道德上是否公正？

+   隐私本身是否是一项基本的好事？或者它是言论自由、和平集会和结社自由的必要条件？

+   我们今天的保障比过去更多还是更少？在过去的几十年里，政府与个人之间的平衡是如何变化的？政府是否拥有更多或更少的监控个人的数据和工具？个人和非政府组织是否有更多或更少的造成伤害（因此需要受到保护）的能力？

+   我们今天的隐私比过去更多还是更少？加密法规在这方面扮演了重要角色吗？

+   在理想世界中，安全和隐私之间的平衡会是什么样子？

+   专注于加密是否是错误的，因为影响隐私和安全的真正问题是所谓的*元数据*？加密技术可以保护这样的元数据吗？即使可以，是否有商业利益去做这件事？

+   反对密码学监管的一个论点是，鉴于密码学的数学不是秘密的，“坏人”总是会找到它。这是一个有效的论点吗？请注意，在枪支控制背景下也有类似的论点。也许“真正的异见者”也能接触到密码学，因此监管将影响大众或“普通”的公民和不太好的公民？

+   禁止使用端到端加密且政府无法访问的法规的实际影响会是什么？

+   Rogaway 认为密码学本质上是政治性的，研究应该承认这一点，并致力于实现有益的政治目标。密码学研究是否失败了公众？还能做些什么？

+   一些密码学（或与密码学相关的）工具是否在本质上存在道德问题？Rogaway 建议这可能适用于完全同态加密和差分隐私 - 你同意吗？

+   什么是最重要的场景，在这些场景中，密码学可以产生积极或消极的影响？大规模恐怖袭击？“普通”犯罪（这些犯罪仍然夺走了比恐怖袭击更多的人的生命）？针对网络基础设施或个人数据的攻击？在压迫政权中的政治异见者？大规模政府或企业监控？

+   与其他国家相比，这些问题在美国有何不同？辩论是否过于以美国为中心？

## 讲座前的阅读：

+   [密码学工作的道德品质](http://web.cs.ucdavis.edu/~rogaway/papers/moral.html) - 请至少阅读第 1-3 部分（脚注版本中的第 1-30 页） - 这篇文章很长，不应不加批判地接受，但它是一篇非常好且发人深省的阅读材料。

+   [“黑暗时代”伯克曼报告](http://cyber.law.harvard.edu/pubrelease/dont-panic/) - 这是由一个委员会撰写的报告，因此（尽管可以争论地说更为冷静）不如 Rogaway 的论文那么吸引人。请至少阅读引言，你也许还会对附录 A 中的个人陈述感兴趣。

+   [数字平衡项目](https://www.digitalequilibriumproject.com/home.html) - 阅读建议 - 这是一个由非常资深现任和前任官员组成的团体，特别是在政府方面，因此他们可能会倾向于更“传统”或“支持监管”的立场。他们的“基础论文”甚至有更多的“委员会撰写”的感觉，但仍值得一读。

+   [加密无政府主义宣言](http://www.activism.net/cypherpunk/crypto-anarchy.html) - 阅读建议 - 尽管听起来更像科幻小说而不是描述实际当前或近未来的现实，但仍然值得一读。

## 案例研究。

由于这样的讨论有时在抽象层面上可能很难进行，让我们考虑一些实际案例：

### 斯诺登揭露

当前关于安全和隐私的辩论的推动力是关于 NSA 对美国及全球公民进行大规模监控的[斯诺登揭露](http://www.theguardian.com/us-news/the-nsa-files)。同时，在众目睽睽之下，苹果、谷歌、Facebook 等公司也在收集大量用户信息。对斯诺登揭露的反击之一是增加对公司支持更强“端到端”加密的压力，例如某些数据不存储在公司服务器上，这些服务器已经变得可疑。我们现在看到一些“反击的反击”，全球执法机构和政府官员试图禁止这种加密技术或强制政府后门。

### 美国联邦调查局与苹果公司案例

我们之前提到过[这个案例](http://www.npr.org/sections/thetwo-way/2016/02/17/467096705/apple-the-fbi-and-iphone-encryption-a-look-at-whats-at-stake)。（我还在[博客](https://windowsontheory.org/2016/03/04/the-iphones-of-terrorists/)上写过关于它。）简要来说，是 FBI 找到了属于圣贝纳迪诺恐怖分子之一的 iPhone。该 iPhone 的内存被一个密钥 $k$ 加密，该密钥由 $H(uid\|passcode)$ 获取，其中 $passcode$ 是用户的六位数字密码，$uid$ 是一个秘密的 $128$ 位密钥，该密钥被硬编码到处理器中。处理器只允许尝试猜测密码十次，然后擦除所有内存。FBI 希望苹果帮助他们创建一个数字签名的软件更新，该更新本质上是在 $10⁶$ 个密码上执行暴力搜索并输出密钥 $k$。该软件更新可以被限制只在该特定 iPhone 上运行。最终，FBI 在没有苹果帮助的情况下成功从 iPhone 中提取了信息。他们使用的方法未知，但可能从处理器中物理提取 $uid$。也可能通过从处理器断开连接或擦除后重写来防止内存被擦除。这样的案例是否会改变你对这个问题的立场？

一些可以提出的问题：

+   考虑到 FBI 对 iPhone 上的信息拥有合法搜查令，苹果拒绝提供所需帮助是否错误？

+   苹果设计 iPhone 使其无法轻易提取信息，这是否错误？他们是否应该确保这些设备在合法搜查令下可以被搜查？

+   如果 FBI 获取信息的唯一途径是获取苹果的主签名密钥（这允许完全破解任何 iPhone，甚至将其变成录音/监控设备），他们这样做是否合适？苹果是否应该设计他们的设备，使其主签名密钥也无法破解？考虑到软件更新对这类设备正常功能至关重要，这是否可能？（最近有报道称[加拿大警方自 2010 年以来一直可以访问 Blackberry 的主解密密钥](http://www.theverge.com/2016/4/14/11434926/blackberry-encryption-master-key-broken-canada-rcmp-surveillance)。）

在圣贝纳迪诺案件中，鉴于两名罪犯都被击毙，没有证据表明他们收到任何帮助，因此质疑破解手机的作用。但有些情况更为复杂。[布里特尼·米尔斯](http://www.npr.org/sections/alltechconsidered/2016/03/30/472302719/mom-asks-who-will-unlock-her-murdered-daughters-iphone)在 2015 年 4 月在路易斯安那州巴吞鲁日被枪杀时 29 岁零 8 个月怀孕。她的孩子通过紧急剖腹产出生，但一周后也去世了。没有强行闯入的迹象，因此她很可能认识她的袭击者。她的家人认为，她谋杀者的身份线索可能在她 iPhone 中，但由于手机被锁定，他们无法提取这些信息。可以想象其他类似的情况。最近一位母亲通过[查找我的 iPhone](http://www.nbcnews.com/news/us-news/find-my-iphone-icloud-lead-cops-kidnapped-teen-n521486)程序找到了她被绑架的女儿。可以想象，解锁手机可能是拯救某人生命的关键。这些情况是否会改变你对上述问题的看法？

### Juniper 后门案例和 OPM 入侵

我们还提到了[Juniper 后门案例](http://www.wired.com/2015/12/juniper-networks-hidden-backdoors-show-the-risk-of-government-backdoors/)。这是一个未知方对 Juniper 网络防火墙的入侵，关键是通过 NSA 声称插入到 Dual EC 伪随机生成器的后门实现的。（参见[这里](https://rpw.sh/blog/2015/12/21/the-backdoored-backdoor/)和[这里](http://blog.cryptographyengineering.com/2015/12/on-juniper-backdoor.html)了解更多。）

由于这次破解的性质，无论谁负责它，都可能在不留下任何痕迹的情况下解密大部分流量，因此我们不知道造成的损害，但此类黑客攻击可能比强迫人们更改信用卡号码具有更严重的后果。当[联邦人事管理局遭到黑客攻击](https://www.lawfareblog.com/why-opm-hack-far-worse-you-imagine)时，数百万已经通过安全审查的人的敏感信息被提取。这包括指纹、从面试和测谎会话中获取的广泛个人信息，以及更多。此类信息可以帮助他们获取更多信息，无论是使用指纹解锁手机，还是利用广泛的社交关系、习惯和兴趣来对特定个人发起非常针对性的攻击以提取信息。

人们可能会问，如果使用更强的加密技术，特别是那些能够使个人控制其数据访问的加密工具，是否能够帮助预防此类攻击。
