# 课程回顾

> 原文：[`intensecrypto.org/public/lec_25_course_recap.html`](https://intensecrypto.org/public/lec_25_course_recap.html)

*发现任何错误/打字错误/令人困惑的解释？[在 GitHub 上打开一个 issue](https://github.com/boazbk/crypto/issues/new)。您也可以在下面评论*

**★ 另请参阅本章的[PDF 版本](https://files.boazbarak.org/crypto/lec_25_course_recap.pdf)（更好的格式/参考文献）★

可能值得回顾一下我们在本课程中学到的东西：

+   也许首先和最重要的是，我们可以*数学地定义*一个密码方案安全意味着什么。在我们研究的情况下，这样的定义可以始终描述为“安全游戏”。也就是说，我们首先定义一个方案*不安全*意味着什么。然后，如果一个方案不是不安全的，那么它就是安全的。不安全的观念是存在某种对抗策略，其成功概率高于应有的概率。我们通常不对对手的*策略*进行限制，而只限制他的或她的*能力*：其计算能力和对系统的访问类型（例如，选择明文、选择密文等）。我们还讨论了*保密性*需要*随机性*，以及许多现实生活中的密码系统失败归结为对随机性来源的错误假设。

+   我们认识到在安全定义中保持*保守*的重要性。例如，尽管选择密文攻击（CCA）安全性的概念似乎过于强大，以至于无法捕捉任何现实场景（例如，我们何时让对手玩解密盒？），但仍然存在许多自然的情况，使用 CPA（选择明文攻击）安全加密而不是 CCA 安全加密会导致对整个协议的攻击。

+   我们看到了如何通过*归约*来证明安全性。假设我们有一个方案 \(S\)，它实现了某种安全性概念 \(X\)（例如，\(S\) 可能是一个实现伪随机生成器安全性概念的函数），我们用它来构建一个我们希望实现安全性概念 \(Y\) 的方案 \(T\)（例如，我们希望 \(T\) 成为消息认证码）。为了证明 \(T\) 是安全的，我们展示了如何将一个在 \(Y\) 的安全游戏中战胜 \(T\) 的对手 \(B\) 转换为在 \(X\) 的安全游戏中战胜 \(S\) 的对手 \(A\)。通常，对手 \(A\) 将运行 \(B\) “在其体内”，为 \(B\) 模拟与 \(T\) 相关的 \(Y\) 的安全游戏。这可能会有些令人困惑，所以请重新阅读最后三句话，并确保您理解这个关键概念。

+   我们还看到了在密码学中我们可以做一些具体而美妙的事情：

+   在*私钥密码学*的世界里，我们看到基于 PRG 猜想，我们可以得到 CPA 安全的私钥加密（特别是密钥短于消息）、伪随机函数、消息认证码、CCA 安全的加密、承诺方案，甚至对于 NP 完全语言的零知识证明。

+   我们看到，假设存在*抗碰撞哈希函数*，我们可以得到密钥短于消息的消息认证码（以及数字签名）。我们讨论了如何将哈希函数建模为*随机预言机*的启发式方法，并使用它来在比特币和密码派生等场景中进行“工作量证明”。 

+   我们还讨论了私钥原语的实际构造，例如 AES 块密码，以及这些块密码如何被建模为伪随机置换，以及我们如何可以通过各种模式（如 CBC 或 GCM）使用它们来获得 CPA 或 CCA 安全的加密。我们还讨论了哈希函数的 Merkle 和 Davis-Meyer 长度扩展构造，以及如何使用 Merkle 树构造进行安全存储。

+   我们看到了革命性的概念——*公钥加密*，两个人可以在事先没有协调的情况下进行交谈。我们看到了基于离散对数（例如，迪菲-赫尔曼协议）、分解（例如，拉宾和 RSA 门限置换）以及*错误学习（LWE）问题*的构造。我们看到了数字签名的概念，并给出了几种不同的构造。我们看到了如何使用数字签名通过证书创建“信任链”，以及保护网络流量的 TLS 协议是如何工作的。

+   我们讨论了一些先进的概念，特别是看到了一个令人惊讶的概念——*全同态加密（FHE）方案*的构造，这个方案被布莱恩·海斯（Bryan Hayes）[正确地称为](http://bit-player.org/2012/computing-with-encrypted-data)“计算机科学中最令人惊叹的魔术之一”。使用 FHE 和无知证明，我们可以得到多方安全计算，这基本上意味着在多个参与者之间的交互协议场景中，我们可以建立一个“虚拟可信第三方”（或者说，我更喜欢称之为“虚拟查克·诺里斯”）。

+   我们还看到了加密的其他变体，如*函数加密*、*证据加密*和*基于身份的加密*，这些加密方式允许“选择性泄露”信息。对于函数加密和证据加密，我们还没有在标准假设下得到干净的构造，而只是在模糊化下，但我们看到了如何使用随机预言机启发式方法和允许高效*配对*操作的群中的离散对数问题的难度假设来获得基于身份的加密。

+   我们讨论了混淆的概念，这可以被视为一个工具，如果存在，将意味着所有其他工具的存在。我们看到了虚拟黑盒混淆不存在，但可能存在一个较弱的称为“不可区分混淆”的概念，并且我们通过见证加密和数字签名方案的例子看到了它的有用性。我们提到（没有证明）它还可以用来获得功能加密方案。

+   我们讨论了量子计算如何改变密码学的格局，使基于格的构造成为我们的主要公钥方案候选。

+   最后，我们讨论了一些在密码学应用中出现的伦理和政策问题，以及密码学现在或未来对社会的影响。

## 我们没有涵盖的一些内容

+   我们没有涵盖可以说是另一条“密码学基本定理”，即单向函数和伪随机生成器的等价性。单向函数是一个高效的映射 \(F:\{0,1\}^*\rightarrow\{0,1\}^*\)，在随机输入上难以逆推。也就是说，对于任何高效的算法 \(A\)，如果 \(A\) 被给定 \(y=F(x)\)，其中 \(x\) 是从 \(\{0,1\}^n\) 中均匀选择的，那么 \(A\) 输出 \(x'\) 且 \(F(x')=y\) 的概率是可以忽略不计的。可以证明单向函数在以下意义上是**最小化**的，即它们对于许多密码学应用是**必要的**，包括伪随机生成器和函数、密钥长度短于消息的加密、哈希函数、消息认证码以及更多。（这些结果中的大多数是通过 Impagliazzo 和 Luby 的工作获得的，他们证明了如果不存在单向函数，那么存在一个**通用后验采样器**，即对于每个将 \(x\) 映射到 \(y\) 的概率过程 \(F\)，存在一个高效的算法，给定 \(y\) 可以从接近 \(x\) 在 \(F(x)=y\) 条件下的后验分布的分布中采样 \(x'\)。这个结果通常被称为标准单向函数和分布性单向函数的等价性。）Hastad、Impagliazzo、Levin 和 Luby 的基本结果是单向函数对于许多私钥密码学也是**充分的**，因为它们暗示了伪随机生成器的存在。

+   与此相关，尽管我们简要提到了这一点，但我们没有深入探讨算法学、启发式、悲观世界、迷你密码学、狂热（以及新的“混淆乌托邦”）的“Impagliazzo 的世界”。如果您对此感兴趣，请阅读[这篇 1995 年的调查](http://www.cs.mun.ca/~kol/courses/6743-w15/papers/russell-fiveworlds.pdf)。

+   我们没有深入探讨私钥加密系统如 AES 的设计。尽管我们讨论了分组密码的操作模式，但我们没有详细描述实践中使用的所有模式。我们也没有讨论如线性分析和差分分析等密码分析技术。我们也没有讨论实践中出现的所有技术问题，例如加密的长度扩展和填充问题。

+   当我们讨论比特币、TLS 协议、双因素认证系统以及一些相当好的隐私方面的内容时，我们只局限于这些系统的抽象，并没有尝试对整个系统进行全面的“端到端”分析。我确实希望你们已经学到了理解这类系统完整运作所需的工具。

+   当我们讨论 Shor 算法时，人们今天实际用来分解数字的算法是*数域筛法*。它及其前身，二次筛法，都值得研究。Shoup 的（免费在线）[书籍](http://www.shoup.net/ntb/)不仅是这些算法的极好来源，也是一般算法群/数论的优秀来源。

+   我们讨论了一些对实际系统的攻击，但还有很多其他攻击能给我们带来重要的教训，不仅关于这些特定的系统，还关于安全和密码学的一般性（以及人类倾向于重复某些类型错误的人类倾向）。

## 我希望你们学到的东西

我希望你们对密码学有了欣赏，并理解了它如何让你惊讶，它不仅能提供惊人的安全特性，还能以微妙但往往具有破坏性的方式失败。在密码学之外，我希望你们从这个课程中学到一点不同的思考方式——足够偏执，能够从对手的角度看待世界，但也得到这样的教训：有时候，如果某件事听起来疯狂但并非完全不可能，它可能就是可行的。

但如果你对这些哲学思考不感兴趣，只要你知道 CPA 和 CCA 的区别，我不会抓到你在重用一次性密码，你应该就很好了 :)

我并没有打算教你们如何实现密码学算法，但我确实希望，如果你们在任何时候需要使用密码学，现在你们有足够的技能去查阅所需内容，并能就现实系统的安全性进行明智的讨论。我也希望你们现在有足够的背景知识，不会被密码学研究论文中的技术术语和形容词的丰富所吓倒，并能查阅你们需要了解的内容，以跟随任何你们感兴趣的文章。

主要来说，我仅仅希望你们喜欢这个学期，觉得这门课程是你们时间的良好利用。我确实是这样认为的。
